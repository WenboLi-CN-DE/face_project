url="http://localhost:8060/vrlMoveLiveness"
前端页面会发送一个包含视频路径、动作列表和阈值配置的JSON请求到后端，后端根据这些信息进行活体检测和动作识别。以下是一个示例请求：

```json


{
    "request_id": "req_20260309_0001_A87B",
    "task_id": "task_vrl_0987654321",
    "video_path": "/data/storage/temp/20260309/user_v_01.mp4",
    "actions": [
        "blink",
        "mouth_open",
        "shake_head"
    ],
    "threshold_config": {
        "liveness_threshold": 0.95,
        "action_threshold": 0.85
    },
    "action_config": {                     // 新增：时间维度配置
        "max_video_duration": 6,          // 比如一个动作2秒，3个动作就是6秒，超过这个时间就不继续分析了
        "per_action_timeout": 2          // 每个动作录制的时间长度
    }
}
```

我们要实现一个后端接口来处理这个请求，进行视频分析并返回结果。以下是一个示例实现：

```json
{
    "code": 0,
    "msg": "success",
    "data": {
        "is_liveness": 1, // 全局最终判定：1为通过，0为不通过
        "liveness_confidence": 0.99,
        "is_face_exist": 1,
        "face_info": {
            "confidence": 0.96,
            "quality_score": 0.88
        },
        "action_verify": {
            "passed": false, // 注意：如果有一个子动作没过，这里即为 false
            "required_actions": ["blink", "mouth_open"], // 本次请求要求的动作序列
            "action_details": [
                {
                    "action": "blink",
                    "passed": true,
                    "confidence": 0.98,
                    "msg": "检测到有效眨眼"
                },
                {
                    "action": "mouth_open",
                    "passed": false, // 该动作未达标
                    "confidence": 0.32,
                    "msg": "动作幅度过小或未检测到张嘴"
                }
            ]
        }
    },
    "request_id": "req_20260309_001",
    "task_id": 1564796632498
}
```

所以，后端需要实现以下功能：
1. 接收并解析前端发送的 JSON 请求。
2. 使用视频路径加载视频并进行人脸检测，判断是否存在人脸以及人脸质量。
3. 进行活体检测，计算活体置信度，得出全局活体判定。
4. 对于每个要求的动作，进行动作识别和分析，计算动作置信度，并与 action_threshold 进行比较，得出每个动作的通过与否。
5. 综合所有动作的结果，得出最终的动作验证结果。
6. 构建并返回一个包含所有检测结果的 JSON 响应给前端。
